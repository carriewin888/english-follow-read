# 英语跟读系统 - 部署指南

## 概述
将英语跟读系统部署到公网，使手机可以正常使用所有功能（跟读、评分、答题、聊天）。

## 部署架构

│  │  hf_score_service    │    │  follow_read.html       │    │

## 第一步：部署后端到 HuggingFace Spaces

### 1.1 创建 Space
1. 访问 https://huggingface.co/spaces（需登录）
2. 点击 "Create new Space"
3. 填写信息：
   - **Space name**: `english-follow-read`
   - **SDK**: 选择 `Docker`
   - **Space hardware**: 选择免费硬件（CPU Basic 即可）
4. 点击 "Create Space"

### 1.2 上传文件
在 Space 的文件页面，上传以下文件：

| 目标文件名 | 源文件 | 说明 |
|-----------|--------|------|
| `Dockerfile` | `hf_Dockerfile` 的内容 | 必须是这个名字 |
| `hf_score_service.py` | 同名的文件 | 后端服务代码 |
| `requirements.txt` | 同名的文件 | Python 依赖 |

### 1.3 等待构建
- HuggingFace 会自动根据 `Dockerfile` 构建镜像
- 首次构建需要 5-15 分钟（需下载 Whisper 模型）
- 构建完成后，访问 Space 页面应显示：
  ```
  {"service":"英语跟读评分服务","version":"3.0","endpoints":["/score","/health"]}
  ```

### 1.4 记录公网地址
- Space 地址格式：`https://你的用户名-english-follow-read.hf.space`
- 例如：`https://xiaoming-english-follow-read.hf.space`

## 第二步：部署前端到 Vercel

### 2.1 准备 GitHub 仓库
```bash
# 在 GitHub 上创建仓库：english-follow-read-frontend
# 然后在本地：
cd D:\BN\future\english
git init
git add follow_read.html quiz.html chat.html
git commit -m "Initial commit: frontend pages"
git remote add origin https://github.com/你的用户名/english-follow-read-frontend.git
git push -u origin main
```

### 2.2 部署到 Vercel
1. 访问 https://vercel.com
2. 使用 GitHub 登录
3. "Add New..." → "Project"
4. 选择刚才创建的 GitHub 仓库
5. 配置：
   - **Framework Preset**: `Other`
   - **Build Command**: 留空
   - **Output Directory**: `.`（根目录）
6. 点击 "Deploy"

### 2.3 记录前端地址
- Vercel 地址格式：`https://english-follow-read-frontend.vercel.app`
- 页面访问地址：
  - 跟读页：`https://english-follow-read-frontend.vercel.app/follow_read.html`
  - 答题页：`https://english-follow-read-frontend.vercel.app/quiz.html`
  - 聊天页：`https://english-follow-read-frontend.vercel.app/chat.html`

## 第三步：配置 n8n 工作流（定时推送）

### 3.1 导入工作流
1. 打开你的 n8n 实例
2. "Create Workflow" → "Import from File"
3. 选择 `english-tutor-agent-v2.json`

### 3.2 修改"格式化推送内容"节点
找到该节点中的 `followReadUrl` 变量，修改为：

```javascript
// 替换为你的实际地址
const BACKEND_URL = "https://你的用户名-english-follow-read.hf.space";
const FRONTEND_URL = "https://english-follow-read-frontend.vercel.app";

const followReadUrl = FRONTEND_URL + `/follow_read.html?title=${encodeURIComponent(topicInfo.topic_emoji + ' 跟读练习')}&date=${encodeURIComponent(topicInfo.date)}&text=${encodeURIComponent(english)}&vocab=${encodeURIComponent(vocabulary)}&patterns=${encodeURIComponent(patterns)}&api_base=${encodeURIComponent(BACKEND_URL)}`;
```

### 3.3 配置 WxPusher 节点
确保 WxPusher 节点配置正确：
```json
{
  "appToken": "AT_vmEhrI8hwfvXhd2CnPm4p4Qacbf0APdc",
  "content": "={{ $json.content }}",
  "contentType": 2,
  "uids": ["UID_XJ0hQ6FIq6Iqlu5hbIOD5Rx3N7lF"]
}
```

### 3.4 测试工作流
1. 点击 "Execute Workflow"（手动触发）
2. 检查微信是否收到推送
3. 点击推送中的链接，确认在手机上能正常打开

## 第四步：手机端测试清单

在手机上测试以下功能是否正常：

### 跟读页面（follow_read.html）
- [ ] 页面能正常加载（HTTPS 连接）
- [ ] 点击"跟读"按钮，能请求麦克风权限
- [ ] 录音后能回听
- [ ] 提交打分后，能收到评分结果

### 答题页面（quiz.html）
- [ ] 页面能正常加载
- [ ] 点击单词能播放发音（Web Speech API）
- [ ] 选择题能正常选择
- [ ] 拼写题能正常输入
- [ ] 提交后能查看成绩

### 聊天页面（chat.html）
- [ ] 页面能正常加载
- [ ] 能发送消息
- [ ] 能收到 AI 回复
- [ ] 能切换聊天模式

## 常见问题排查

### 问题1：HuggingFace Space 构建失败
- 检查 `Dockerfile` 是否命名为 `Dockerfile`（区分大小写）
- 检查 `requirements.txt` 中的依赖是否正确
- 查看 Space 的 "Settings" → "Factory replica settings" → "View logs"

### 问题2：前端页面无法访问后端
- 检查 `follow_read.html` 中的 `API_BASE` 是否正确
- 确认 HuggingFace Space 已正常运行
- 检查浏览器控制台是否有 CORS 错误

### 问题3：微信中无法打开链接
- 确认前端页面使用 HTTPS（Vercel 自动提供）
- 确认链接没有使用 localhost 或内网 IP

## 维护

### 更新后端代码
1. 在 HuggingFace Space 的文件页面直接编辑 `hf_score_service.py`
2. 或者本地修改后推送更新（如果连接了 Git）

### 更新前端页面
1. 修改本地 HTML 文件
2. 推送到 GitHub
3. Vercel 会自动重新部署

### 查看评分服务日志
- 在 HuggingFace Space 页面点击 "Pricing and logs" → "Logs" 查看实时日志
